# What are motion graphics?

Motion graphics refers to the type of digital art where graphic design elements are animated (like moving, rotating, or changing color).

We encounter motion graphics in our daily lives. Whether it‚Äôs animated elements on a website, or watching the Duolingo bird scold us ‚Äî we‚Äôre surrounded by shapes and colors in motion.

# Motion Graphics in Blender? The 3D app?

Blender has been quickly becoming more popular to achieve 2D styles, like the anime cel shading or outline techniques that bring graphic elements to the 3D scene.

But even before that, Blender has been capable of motion graphics. It has a robust suite of tools that can be used for animating - like typography rendering, keyframes, video sequencing, even compositing.

This article won‚Äôt be a comprehensive exploration of all these techniques. It‚Äôll be a more realistic workflow where I use other apps in the flow as well, like a 2D vector app for creating quick textures, or a video editing app for compiling and color correcting animations. But many of these process can be accomplished inside Blender itself, it‚Äôs just a bit cumbersome to be honest.

# How to make motion graphics in Blender?

Like I mentioned above, motion graphics are just shapes and text, which Blender has plenty of primitives for! I recommend starting with something simple, like creating text and animating it using keyframes.

## Shapes

For shapes, try using a Plane for squares and rectangles. Need a circle? Use a Circle Mesh (since it‚Äôs filled in).

![The Add dropdown with the top available mesh objects (like plane or circle)](image%201.png)

Need something more complex? You can always draw it in a vector based app, export an SVG, and import that as the basis. Or manually draw it using Blender‚Äôs internal Bezier Curve tool (which isn‚Äôt too bad, especially when working in the 3rd dimension).

![The SVG import option highlighted in the File menu dropdown](image%202.png)

### Stroked elements

Need a stroke or outline for your shape? This is a bit tricky in Blender (and starts to show it‚Äôs limitations in a way). There‚Äôs a few ways to handle this.

The first way is the **Wireframe modifier**. Duplicate your object and add a Wireframe modifier to your shape. This works great if it‚Äôs an ‚ÄúN-gon‚Äù (meaning no ‚Äúpolygons‚Äù or triangles breaking up your mesh). This works best with a Plane for example.

![A stroked element made with a Plane mesh and a Wireframe modifier on it](image%203.png)

The second way is [the Solidify modifier](https://youtu.be/cnQu1kMs49s?t=175). It basically duplicates your mesh, makes it slightly larger, and gives you an impression of an outline. This technique is better if you‚Äôre trying to make a more complex 3D shape look 2D and cartoony.

Another approach would be to use Blender‚Äôs materials combined with an SDF algorithm to create your shape‚Ä¶

And obviously the simplest approach would be to export the UV texture map and add the stroke in a third party app like Figma or Photoshop.

As you can see, stroked elements are a little tricker to handle and definitely aren‚Äôt as easy as other apps.

> üåê My preferred method is to use geometry nodes and convert the mesh to a curve, then use a Curve Profile to basically create my own ‚ÄúWireframe modifier‚Äù. This also makes it easy to ‚Äúduplicate‚Äù the object since you can just use a Join Geometry node with the original object and the new wireframe stroke object.

## Typography

Blender supports adding 2D text to a scene, and you can even easily extrude the text to make it 3D.

Find a 3D Viewport window in your app, find the toolbar on the top, and go to Add > Text. Then you can go to the Text properties tab in the Properties window (usually bottom right of your screen, looks like a cursive lowercase `a` icon).

![The Text properties panel](image%204.png)

You can control the font family here, as well as the size, and even alignment. It‚Äôs pretty versatile.

> üÖ∞Ô∏è I will note that some fonts work better than others. Anything too complex, like calligraphy or fonts with ‚Äútexture‚Äù may not render correctly. It‚Äôs recommended that you always fill font‚Äôs as n-gon‚Äôs - this ensures that the font retain‚Äôs it‚Äôs shape the most like an SVG.

## The ‚ÄúArtboard‚Äù

Blender is probably similar to After Effects, where your viewport what the camera is seeing. In Blender, you can have multiple cameras, but only one is enabled at one time.

The size of the camera‚Äôs viewport is defined by the ‚Äúresolution‚Äù you set in the Output Settings.

![The Output Settings panel with the Format section expanded to show Resolution](image%205.png)

So if you set it the X to 1920 and Y to 1080 it‚Äôd look like a classic widescreen size. And if you change it to 3840 x 2160, nothing would visually happen ‚Äî but when you ‚Äúrender‚Äù your image later it‚Äôll be larger.

> üìÅ I recommend checking out [my Blender plugin **‚ÄúRender Buddy‚Äù**](https://github.com/whoisryosuke/blender-render-buddy). It helps you quickly set your camera to standard social media dimensions like 1080p widescreen or IG Story format.

But if you change your resolution to say, 1080 by 1920, the viewport isn‚Äôt going to change how you‚Äôd expect in an app like Photoshop and crop evenly from both sides. You‚Äôll often have to move the camera to get the right composition again.

> üé• This is probably the most frustrating part of working within Blender. I‚Äôll often have a camera for each resolution I‚Äôm working with (e.g. `Camera.Square` or `Camera.Widescreen`)

This isn‚Äôt the case for camera‚Äôs set to an Orthographic view (instead of the default Perspective).

## Animation

Animation happens in Blender using keyframes. You define the length of your animation by setting the start and end frame of the ‚Äútimeline‚Äù (similar to most animation apps).

![The Timeline window with various keyframes. Some keyframes are connected with orange rectangles. ](image%206.png)

You can set keyframes on various properties of objects, like it‚Äôs position, rotation, or scale. You set these keyframes by right clicking on the property you want (like the Z position) and selecting ‚ÄúInsert Keyframe‚Äù. Or you can activate the Auto Keying (the ‚Äúrecord‚Äù looking icon in the Timeline left of the play button) and do anything to the object to record it as a keyframe.

![The right click context menu for the Location properties on an object.](image%207.png)

Then Blender can interpolate between these keyframes to animate - even using easing methods like Quadratic. Which are all editable using the Graph Editor, giving your granular control over each keyframe.

For animations that include characters, you can also attach bones to your objects and animate those bones.

And there‚Äôs plenty of other small tools in the toolbox - like generating keyframes with noise algorithms (like an ‚Äúearthquake‚Äù effect), or actions which bundle keyframes into reusable groups (think like a video game character‚Äôs ‚Äújump‚Äù or ‚Äúrun‚Äù animation - these would be actions).

Primarily in this article I‚Äôll be covering animation using geometry nodes, but if that ever gets intimidating know that there‚Äôs plenty of alternatives that are more visual and less math.

## ‚ÄúSaving‚Äù your animation

This is where Blender differs from a lot of applications. Blender has a process of ‚Äúrendering‚Äù individual frames. For an animation that‚Äôs set to 24 frames per second, it‚Äôll render exactly that many‚Ä¶per second.

You can have Blender save these frames as images, or compile them into a video for you (like After Effects would).

I personally render to images - either JPG/PNG for smaller projects, and EXR for professional projects that need color correction in post.

![A folder full of animation frames as PNG files incrementing from 0.](image%208.png)

Then I import the image sequence into Premiere Pro or Da Vinci which generates a video.

![Adobe Premiere Pro with my Joycon 2 Piano animation project.](image%209.png)

The key thing to note here is the ‚Äú**rendering**‚Äù part. Blender has multiple renderers available. The default, **Eevee**, is the **best for animation** since it renders the fastest. But you can also use the **Cycles** renderer, which is a bit slower, but gives you more **photo realistic** renders and unlocks different advanced techniques (like refractive glass, or custom shadow mapping).

# Motion Graphics with Geometry Nodes

Now that we understand the elements of a motion graphic and how to make them in Blender, let‚Äôs do all of that process ‚Äî but inside of geometry nodes.

Geometry nodes are ‚Äúnodes‚Äù on a ‚Äúnode graph‚Äù that generate ‚Äúgeometry‚Äù. Basically it‚Äôs a visual scripting language that lets you chain commands like you would writing a script in a programming language. There are nodes that represent various bits of data, like the actual geometry itself - or it‚Äôs position in 3D space as a ‚Äúvector‚Äù.

It‚Äôs a lot to take in, so I‚Äôd definitely recommend watching a few tutorials - but I‚Äôll try to take it slow so hopefully you‚Äôll be able to pick up a few things here.

## Making shapes

Like I mentioned before, if you want a box or rectangle you‚Äôre probably looking for a Plane object.

In geometry nodes, you‚Äôll find this under the Grid node. It‚Äôs basically the same concept, and you can control the number of subdivisions (or cuts) the plane has.

![A node graph with a Grid node connected to output and a Plane visible in the 3D viewport](image%2010.png)

If you want a Circle, you can use the Circle Mesh and set the fill type to N-Gon. You can use the Curve Circle too - but it‚Äôll require you to attach a ‚ÄúFill Curve‚Äù node afterwards or it won‚Äôt render anything.

![A node graph with a Circle Mesh and Curve Circle nodes connected to output and a Circle visible in the 3D viewport](image%2011.png)

### Rounded Rectangles

To make a rounded rectangle, you can apply a **Bevel modifier** to a Plane. If you change the Bevel mode to **Vertices**, it‚Äôll target the corners of the Plane and round them. From there you can decide how smooth you want the corner by changing the **Segments**.

![A Plane object turned into a rounded rectangle with a Bevel modifier visible in the sidebar on right](image%2012.png)

You can also use the Bevel tool in edit mode to achieve a similar effect, but this lets you edit the bevel later or animate it if you want.

For Geometry Nodes, I created a custom node group for creating a rounded rectangle. It takes a width and height, as well as the radius of the corners (and their ‚Äúdetail‚Äù using the count property).

![Rounded Rectangle custom node group with Width, Height, Count, and Radius input properties.](image%2013.png)

This node group is made up of a Quadrilateral node that makes a plane as a curve. Then that gets rounded using a Fillet Curve using the radius and detail count we provide. Then we fill the curve to make it visible during rendering.

![4 UI cards and a Node Graph for the Rounded Rectangle custom node group](image%2014.png)

This is a really handy node in the toolbox, you can easily extend it to create components like the cards above.

## Animation

There are a multitude of ways you can handle animation in geometry nodes. Let‚Äôs start with the basics.

### Creating an animation

Most animations are powered by the current frame number. This is accessible from the ‚ÄúScene Time‚Äù node, which has the time in seconds and frames.

![Node graph using the Scene Time node to drive an animation using the current frame number](image%2015.png)

We can use the current frame number to drive our animations. In this case, we use a Set Position node to offset our geometry on the Z axis by the current frame number. So it basically rises the longer the animation plays.

### Animation Interpolation

So now that we understand how to drive our animations, let‚Äôs see how we can control some animations.

Normally when working with animations, it‚Äôs easier to make the animated number go from `0` to `1`. The `0` represents the beginning of the animation, and the `1` represents the end. This makes it easier than say working with frame numbers, like `0` to `180` when you need to do math operations.

I like to make an Integer node that holds the ‚Äútotal frames‚Äù of my animation. The frame count gets divided by these total frames, and it gives us that `0` to `1` number we‚Äôre looking for (e.g. we have `180` total frames, if we‚Äôre at frame `180` it‚Äôs `180/180` ‚Äî making it `1`).

![A node graph showing how to use the current frame number and a total frames number to drive an animation between two points.](image%2016.png)

> ‚ôªÔ∏è Protip: You can make your Total Frames node into a ‚Äúnode group‚Äù so you can reuse it in different places (but edit it in one place).

If you‚Äôre not into math, or not vibing with the node graphs, you might not understand how this equation works. Luckily we can use another node to illustrate this concept better -and it works functionally the exact same way (and it‚Äôs useful in a different way ‚Äî you‚Äôll see).

![Similar to the last node graph, but using the Map Range node instead of the Math Division node for deriving a number between 0 and 1.](image%2017.png)

Using the Map Range node, we can do the same process of ‚Äúmapping‚Äù our start frame (`0`) and end frame (`180`) to `0` and `1`. This takes the current frame number and checks where it falls along the ‚ÄúFrom‚Äù min and max. Think of it like finding the progress we‚Äôve made in that number range as a percentage ‚Äî then using that percent to figure out where we are in a different number range. So if we‚Äôre at frame `90` of `180` we‚Äôre 50% done ‚Äî which means we‚Äôre at `0.5` (50% of 1).

![Diagram for animation progress as progress bar measured in percents. Above is a timeline for frames in the animation, showing the total frames on right. And below shows how the percent maps from 0-100% to 0-1.](Map_Range_Explained.png)

Cool, so why do we want our animation to go from `0` to `1` again? Why was this so important?

It lets us do cool stuff like this:

![Node graph for an Wave texture animation powered by the Scene Time mapping to the Phase Offset of the Wave node.](image%2018.png)

We add another Map Range node and take our number range from `0-1` to `0 - 2pi`. This lets us animate a number through the circumference of a circle - which is handy for the Wave Texture node which has a Phase Offset property repeats the wave animation every `2pi`. So this way you get 1 exact wave animation.

But ultimately you get the idea. You want an the animation to be an easy number to do math with.

### Looping animations

There are lots of techniques to get something to ‚Äúloop‚Äù or seamlessly repeat when played back to back.

<BlueskyEmbed id="3lmcux5cabv2i" />

**Ping Pong**

The biggest tool you can use is the Math node with the Ping Pong setting. This takes a number and limits it by another number. Say you limit by 5, your frame would increment like `1,2,3,4,5,4,3,2,1` and keep repeating. This lets you have an animation ‚Äúping pong‚Äù or start and finish - then go back to start.

Here‚Äôs how that‚Äôd look as nodes:

![Node graph for looping an animation using the Scene Time and a Math Ping Pong node.](image%2019.png)

We take our total frames and cut them in half, so our animation will go from `0-1` to the halfway point, then reverse, and go from `1-0`. This will give us a number that perfectly loops - every time.

**Wave Texture**

Another great way to achieve a loop is the wave texture. If you animate the Phase Offset property from `0` to `2*pi` you‚Äôll get 1 full ‚Äúwave‚Äù animation. So if you multiply `2*pi` by the number of loops you want, you can get the animation to repeat.

![Node graph for looping an animation using the Wave Texture node.](image%2020.png)

### Mixing

This technique involves taking 2 animations and ‚Äútweening‚Äù or interpolating between them using the Mix node. The Mix node allows you to take 2 values and interpolate between them based on the factor your provide (see where the `0` to `1` kinda math comes in?).

This one is great if you want to loop but you don‚Äôt want it to look as obvious, and you need to work with non-repeating techniques (like Noise instead of Wave texture nodes). I picked this technique up from Ducky3D on YouTube.

![Node graph with the Scene Time powering a Mix node between 2 different Wave Texture nodes.](image%2021.png)

Here we have 2 Noise texture nodes offsetting our object. We‚Äôll animate from one another smoothly ‚Äî then backwards with the ping pong effect shown above.

### Easing

The last bit of basics I‚Äôll share is easing in geometry nodes. You can apply easing to any animation by using the Float Curve node. This lets you draw the easing curve.

> ‚§¥Ô∏è If you don‚Äôt know what an easing curve is, it‚Äôs basically a graphical representation of a formula (like `x = y` which is linear interpolation). The curved line essentially tells our animation to slow down, speed up, or even reverse. There‚Äôs plenty of websites out there that illustrate various popular easing curves (like ‚Äúease in‚Äù or ‚Äúease out‚Äù).

![Node graph for animation with easing using the Float Curve node, which has a couple points inside forming a S shaped curve.](image%2022.png)

Here the Float Curve node will speed up the animation a bit then slow it down.

<BlueskyEmbed id="3llc6voeept26" />

The most important part to note here is that the Float Curve works using a value from `0` to `1`. So if you make the number bigger than that - it‚Äôll ‚Äúclamp‚Äù it (meaning stop at highest value - in this case always `1`). This is another reason why it‚Äôs important to map animation time to `0-1` range.

> ‚ôªÔ∏è I usually make node groups with my favorite Float Curve nodes so they‚Äôre more easily reusable across different geo nodes or within the same graph (or for adding to your asset library!).

### Animating opacity

Getting an object to fade in and out is kinda simple, albeit tedious depending on what you need to do.

<BlueskyEmbed id="3lnilybrinx2b" />

Everything in Blender works using the concept of Materials. 3D objects have ‚Äúmaterials‚Äù applied to them, and that‚Äôs what tells the renderer how to interpret the object. Whether that‚Äôs making the object a different color - like red or blue, or adding an image as a ‚Äútexture‚Äù. So when we want to change something about an object‚Äôs appearance, we need to change it‚Äôs material.

Materials in Blender are conveyed as node graphs, similar to geometry nodes. If you go to the Shading tab and create a new material, you can see a new graph popup - usually with a Principled BSDF node.

![Blender with the Shading tab selected. A 3D viewport is on top, with a shader node graph on bottom with a Principled BSDF node connected to the Surface output.](image%2023.png)

When we talk about opacity in Blender, we‚Äôre usually talking about the Alpha property of the Principled BSDF node (inside of our material). By default you might not see the alpha property changing, you might need to go into the Viewport Shading mode (the 4 circle icons in top right of 3D viewport, click the 3rd or 4th one).

But how do we animate this?

We‚Äôll use Blender‚Äôs Store Named Attribute node. This node lets us store data on our mesh (or instances). Like any data. In our case we want to pass the animation value we‚Äôve been deriving from the frame number. We‚Äôll name the attribute `opacity` so it‚Äôs clear what‚Äôs it‚Äôs for.

![A geometry node graph with the Store Named Attribute node connected to an animation derived from Scene Time](image%2024.png)

Now the important part, we need to get the attribute we stored in the geometry nodes and use inside our object‚Äôs material. Using an Attribute node, if we set the name to `opacity` like we did before, we can access the value we saved.

![The shader node graph with an Attribute node connected to the Alpha input of the Principled BSDF](image%2025.png)

Since our value maps from `0` to `1` it works perfectly for the Alpha property. `0` means fully transparent and `1` means it‚Äôs visible.

> ‚ö†Ô∏è This technique works great, but if you have multiple objects with their own materials (like 5 different images), you‚Äôll probably need to do this setup in **every material** to get the opacity animation. You can get clever and create a single material and offset the texture - but that requires a bit of setup too.

## Advanced Animations

That‚Äôs the basics, where do we go from here? Let‚Äôs get into more advanced techniques.

### Animating Instances 101

This is where things get interesting. In Blender you can take points on a mesh and replace them with ‚Äúinstances‚Äù of another object. So if you had a sphere with a bunch of vertices, you could convert those into instances of say‚Ä¶a water droplet.

![A node graph with a UV Sphere node that get turned into a wireframe and combined with an Instances on Points node that spreads small Ico Sphere on the original sphere‚Äôs surface.](image%2026.png)

For motion graphics, this is a cool technique in general. I like to just take 3D shapes and place primitives on them. You can achieve some old school graphic type aesthetics pretty easily.

So how does this work with animation?

We can use the Index node to access the each instance‚Äôs unique number (incrementing from 0 to however many instances we have). The index allows us to alter the animation per each object, giving us access to a unique number that isn‚Äôt random on each instance.

Here‚Äôs a simple example you can try. Create some instances, then add a Wave Text node and connect it to a Combine XYZ node that goes to a Set Position node.

This will animate all the instances at different rates in a wave like pattern.

Now create a Vector node and connect it to the Wave Texture Vector input.

![A node graph with an animation powered by a Wave Texture. A Vector node that‚Äôs zeroed out is connected to the Wave‚Äôs Vector input.](image%2027.png)

Notice how this animates all the instances at the same time? That‚Äôs because normally the default value for the Wave texture Vector is the object‚Äôs position. You can test this by adding a Position node and connecting it to the Vector input of the Wave Texture. Nothing should happen, because it‚Äôs the same number. The position provides a different ‚Äústarting point‚Äù for each instance, providing a new wave animation value. When we provide a `0,0,0` point for every instance, they all animate the same without their unique position.

![The Position node connected to the Wave Texture‚Äôs Vector input.](image%2028.png)

Now try adding an Index node and linking it to the Vector input (left side) of the Wave texture node.

Now you should have a wave animation that‚Äôs unique to each instance.

![The Index node connected to the Wave Texture‚Äôs Vector input.](image%2029.png)

This is a great way to create a ‚Äúfaux‚Äù sine wave using a Line curve.

![One of my artworks made of stacked lines curved into varying sine waves. The lines go from blue to green from top to bottom.](GN_-_Faux_Wireframe_-_R1_001.png)

### Animating Each Instance

What if we want to animate each instance with it‚Äôs own animation ‚Äî like a ‚Äúfade in‚Äù animation. We know we can access each instances index, so how can we use this to determine it‚Äôs animation time?

If we know that our animation is a certain length - say `180` frames, and we know how many instances we have, then we can figure out how long 1 ‚Äútime segment‚Äù is for each instance. For example, **10 instances** would have **18 frames of animation time each**.

```jsx
// This is what we're doing with the nodes
const instanceCount = 10;
const totalFrames = 180;
const timeSegment = totalFrames / instanceCount;
```

And since we have the index of each instance, we can multiply the index by the animation time to figure out what time segment we‚Äôre ‚Äúinside‚Äù of. For example, if we‚Äôre on the 3rd instance, our animation would start at frame `54` or `18 frames * 3`.

```jsx
// This is what Blender is basically doing with Instance on Points
for (var index = 0; index < instanceCount; index++) {
  const currentTimeSegment = timeSegment * index;
}
```

And we know that each time length (in this case) is `18` frames, so if we know the start frame for each instance, and the length, we can determine the start and end times basically. And with that - we can use a Map Range node to go ‚ÄúFrom‚Äù the start/end time to `0-1`.

```jsx
const start = currentTimeSegment;
const end = currentTimeSegment + timeSegment;

// This `map()` function works like the Map Range node, converting one number range to another
// It's very common in shader code (like GLSL) or graphics libraries
const animation = map(frameNumber, start, end, 0, 1);
```

For this example, let‚Äôs use a line because it‚Äôs the easiest to visualize. In this example I‚Äôm also going to use a Collection for my instances, so I can use premade graphics I have saved.

![A geometry node graph that creates instances on a line with no spacing between them, and a large frame with an animation for each instance](image%2030.png)

Here we have a Mesh Line that has the same number of points as the number of instances in our collection (handy). Next we place instances on that line‚Äôs points, and each instance is an object from a collection (in this case different ‚ÄúUI cards‚Äù).

Then the magic happens (mostly) in a Set Position node. I get the time segment using the instance count and the total frames (like I described above), then we multiply that by the index. Here you can see that I also offset the index by `1` because it starts at `0` usually and that throws the math off a bit.

### Enter and Exit Animations

In animation, you often have a classic ‚Äúenter‚Äù and ‚Äúexit‚Äù animation that act as transitions for an object in a scene. You might be familiar with this in a lot of websites and apps, where objects ‚Äúfade in‚Äù when they load and then ‚Äúfade out‚Äù when they leave.

With our current setup, animate things using a value from `0-1` that‚Äôs based on the current frame number in relation to the total frames. If we want to have an enter and exit animation, they just need to happen in the first and second halves (`0-0.5` and `0.5-1`).

```jsx
const enterAnimation = map(animation, 0, 0.5, 0, 1);
const exitAnimation = map(animation, 0.6, 1, 0, 1);
```

This is similar to the ping pong technique I showed earlier, but you have control over the reverse and it‚Äôs not just a mirror of the first animation.

We can imitate this effect in Blender pretty easily using a combination of the Map Range nodes and a Switch node.

![A geometry node graph with a Switch node connected to a Set Position node's offset. The Switch node toggles between the enter and exit animations, each their own Map Range node](image%2031.png)

The Switch node does the magic here. We check if the animation number (`0-1`) is greater than `0.5` ‚Äî if it is, we return the Exit animation. If not, we do the Enter animation. So ideally it starts with the enter animation, the clicks on the exit animation.

> üßä In this setup I go from `0-0.4` for the enter animation to give the object a slice of time to just stay around before it leaves.

I animate the object‚Äôs Z position here from `0` to `1` and `1` to `0` back again. This moves it up then down again.

But I also use a Store Attribute node to save an `opacity` attribute I can use in my material to ‚Äúfade in‚Äù and ‚Äúfade out‚Äù the object while it moves at the same time.

# Tips and tricks

## Stacking elements

Another caveat of using Blender is that it‚Äôs not simple to layer elements. Normally when you work in Photoshop and Figma you‚Äôd drop down a rectangle, maybe some text, and each element would stack in layers.

For Blender, you‚Äôll find that if 2 items occupy the same space there‚Äôll be visual aberrations due to an effect commonly referred to as [**‚Äúz-fighting‚Äù**](https://en.wikipedia.org/wiki/Z-fighting). Try making a Plane, then make another Plane right on top of it. You should see that they ‚Äúfight‚Äù each other for visibility, tearing between one and the other.

The way to avoid this is to simply offset each element on the Z axis (or whichever axis your elements are stacking on). I recommend using tiny increments like `0.0001`.

## Skip the lighting (and BSDF)

When you‚Äôre rendering 2D animations that are flat and don‚Äôt require shadows or 3D-style shading, I recommend making a material where you connect your color or texture directly into the Output (instead of going through the Principled BSDF node first).

![A RGB node connected directly to the Output in the Shader graph.](image%2032.png)

This setup allows you to **skip the lighting phase** and go directly into rendering. Since you‚Äôre not using any shaders that use light data, you don‚Äôt need any in your scene. No more having to throw down some lights because everything looks black - all the colors will look accurate all the time.

This works great in combination with making the background transparent.

> ‚ö†Ô∏è Be aware however that you won‚Äôt be able to use **opacity** (or ‚ÄúAlpha‚Äù in Blenders terms) with this setup. Only flat colors work - no fading in and out. If you need opacity, you‚Äôll have to use Principled BSDF (and light your scene ü•≤).

## Get that Dribbble look

Use the Orthographic camera setting for that classic ‚Äúperfectly flat‚Äù look. Perspective camera has it‚Äôs uses for creating more dynamic shapes and compositions, but most motion graphics are done in an orthographic view to keep things uniform across the canvas.

![The Properties panel on the Camera tab with the Lens Type dropdown expanded and the Orthographic option selected.](image%2033.png)

## Transparency

Use Transparency to your advantage. Eevee and Cycles both support rendering transparent backgrounds.

Go to the Render Properties tab in the Properties panel (usually right sidebar). Under Film there should be a checkbox for Transparency for both Eevee and Cycles.

![The Render Properties tab with the Film section expanded on bottom and Transparency checked.](image%2034.png)

Save to a PNG and then stack the animation however you need. This is great for layering animations - as well as just changing up the background color.

## Parents = Groups

Parent objects to an empty object if you need to animate multiple elements at once. Great for creating ‚Äúcomponents‚Äù like combining a plane, a few text layers, and even an SVG icon as a curve.

1. Create an Empty object (any type works, I usually use Arrows or Cube)
2. Select all the objects you want to group together.

   You can select multiple objects by holding CTRL/CMD on your keyboard and clicking another object in the 3D Viewport or Outliner (the panel with the list of all objects).

3. Now also select the Empty object we just made. It should be the last thing you select.
4. Right click inside a 3D viewport and go to Parent > Object.

   ![The right click menu from the 3D Viewport with the Parent menu option selected and Object selected in the sub dropdown](image%2035.png)

5. All your objects should be placed ‚Äúunderneath‚Äù the Empty object in the Outliner window.

Now when you want to move all the objects together, just select the Empty object and move it instead. Everything inside will move (or rotate, scale, etc) too.

I usually name the Empty object whatever the group represents (like `PianoKeys` when I have multiple piano keys objects).
